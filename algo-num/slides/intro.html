What is an Algorithm?

  • Recipe to compute a task

    • Well-defined
      Un-ambiguous / easy to interpret

    • Elementary
      Easy to carry out

    Depends on computational model

  • Eg. Addition (by counting)

    3(1+1+1)+4(1+1+1+1)=7\begin{aligned} &\quad 3 && \step{1}{(1 + 1 + 1)}\\ + &\quad 4 && \step{2}{(1 + 1 + 1 + 1)}\\ \hline = &\quad \step{3}{7} \end{aligned}+=​347​​(1+1+1)(1+1+1+1)​​

  • Algorithm add(a, b)

    1. Create 2 counters: result, op
    2. Increment both till op equals a
    3. Reset op to 0
    4. Increment both till op equals b
    5. Output result

    • Intuitive
      Follows from definition

    • Not easy on large numbers

Why Algorithms?

  • Automation

    • Quickly compute a task

    • Using limited resources
      eg. disk space

    • Parallelize a task

    • Security

    This talk: focus on time complexity

  • Computational Model

    • 64-bit registers

    • Arithmetic ops: add, mul, div

    • Logic ops: and, or, not

    • Control flow: jump, if, etc.

    • Random-access memory
